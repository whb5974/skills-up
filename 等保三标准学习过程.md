## 三、学习内容摘要

### 3.1 GB/T 22239-2019 学习要点

**核心内容：** 规定了各等级系统的安全要求

**安全要求分类：**
```
技术要求（5 个层面）：
├── 安全物理环境（10 个控制点）
├── 安全通信网络（8 个控制点）
├── 安全区域边界（12 个控制点）
├── 安全计算环境（15 个控制点）
└── 安全管理中心（5 个控制点）

管理要求（5 个层面）：
├── 安全管理制度（8 个控制点）
├── 安全管理机构（10 个控制点）
├── 安全管理人员（8 个控制点）
├── 安全建设管理（12 个控制点）
└── 安全运维管理（15 个控制点）
```

**等级划分：**
- 第一级：自主保护级
- 第二级：指导保护级
- 第三级：监督保护级
- 第四级：强制保护级
- 第五级：专控保护级

**扩展要求：**
- 云计算安全扩展要求
- 移动互联安全扩展要求
- 物联网安全扩展要求
- 工业控制系统安全扩展要求

### 3.2 GB/T 28448-2019 学习要点

**核心内容：** 规定了测评内容、方法、判定规则

**测评方法：**
| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 访谈 | 与相关人员交流 | 管理类要求 |
| 检查 | 查看文档、配置、记录 | 文档类、配置类要求 |
| 测试 | 实际操作验证安全功能 | 技术类要求 |

**测评对象：**
- 网络设备
- 安全设备
- 服务器/操作系统
- 数据库
- 应用系统
- 管理制度文档
- 人员

**判定规则：**
- 单项判定：符合/不符合
- 组合判定：AND/OR 逻辑
- 加权判定：按权重计算得分
- 否决项：不符合则整体不通过

### 3.3 GB/T 28449-2018 学习要点

**核心内容：** 规定了测评工作的流程和方法

**测评流程四阶段：**
```
1. 测评准备 → 2. 方案编制 → 3. 现场测评 → 4. 分析与报告
```

**各阶段核心任务：**

**阶段 1：测评准备**
- 确定测评范围
- 签订测评合同
- 组建测评团队
- 准备测评工具

**阶段 2：方案编制**
- 收集被测系统信息
- 确定测评指标
- 设计测评用例
- 编制测评方案

**阶段 3：现场测评**
- 召开首次会议
- 执行测评用例
- 记录测评结果
- 确认测评发现

**阶段 4：分析与报告**
- 汇总测评结果
- 进行风险分析
- 编制测评报告
- 召开末次会议

**报告要求：**
- 8 个标准章节
- 三级审批流程
- 电子签章
